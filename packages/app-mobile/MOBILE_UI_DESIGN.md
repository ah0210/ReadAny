# ReadAny Mobile UI 设计文档

## 一、设计原则

1. **移动优先，非桌面缩小** — 每个页面针对触屏单手操作重新设计，而非桌面版等比缩放
2. **拇指区优先** — 核心操作放在屏幕下半部（底部导航栏、底部弹出面板）
3. **全屏沉浸阅读** — 阅读器默认全屏，轻触唤出控制层
4. **渐进式功能暴露** — 首屏精简，高级功能（RAG、向量化）通过设置页面入口到达
5. **手势驱动** — 左右滑动翻页、下拉返回、边缘滑动唤出面板

---

## 二、桌面端 vs 移动端功能对照

| 桌面端功能 | 移动端策略 |
|-----------|-----------|
| TabBar（顶部多标签） | ❌ 移除，移动端一次只看一本书 |
| Sidebar（左侧导航） | → 底部 Tab 导航栏 |
| CommandPalette（Cmd+Shift+P） | ❌ 移除（移动端无键盘快捷键需求） |
| 三栏式阅读器（笔记+阅读+AI） | → 全屏阅读 + 底部弹出 Sheet |
| 鼠标悬浮 Tooltip | → 长按提示 / 直接展示文字标签 |
| 右键菜单 | → 长按菜单 |
| 拖放导入文件 | → 系统分享 / 文件选择器 |
| 阅读器浮动工具栏（顶部） | → 轻触唤出顶部+底部工具栏 |
| 桌面端 Dialog 弹窗 | → 全屏页面 或 底部 Sheet |
| 阅读统计热力图（宽屏） | → 精简卡片 + 可滚动热力图 |

## 三、导航架构

### 底部 Tab 导航（5 个入口）

```
┌──────────────────────────────────────────┐
│                                          │
│            [当前页面内容]                  │
│                                          │
├────────┬────────┬────────┬────────┬──────┤
│  📚    │  💬    │  📝    │  🧩    │  👤  │
│ 书库   │  AI    │  笔记  │  技能  │  我的 │
└────────┴────────┴────────┴────────┴──────┘
```

| Tab | 对应桌面端 | 说明 |
|-----|-----------|------|
| 书库 | HomePage + Sidebar/Library | 书籍网格 + 标签筛选 |
| AI | ChatPage | 独立 AI 对话 |
| 笔记 | NotesPage | 全局笔记管理 |
| 技能 | Skills 页面 | AI 技能管理（内置 + 自定义） |
| 我的 | Settings + Stats | 阅读统计 + 设置入口 |

**说明**：阅读器是一个独立的全屏页面，从书库点击书籍进入，不在 Tab 导航中。

### 4.2 阅读器页（全屏，非 Tab 页）

阅读器是本应用最核心的页面，采用 **全屏沉浸 + 轻触唤出** 模式。

#### 默认状态（沉浸阅读）

```
┌────────────────────────────────────┐
│                                    │
│                                    │
│                                    │
│          [书籍内容全屏]              │
│          FoliateViewer             │
│                                    │
│                                    │
│                                    │
│                                    │
└────────────────────────────────────┘
```

- 无任何 UI 覆盖，纯内容
- **左右滑动** 或 **点击左右边缘** → 翻页
- **轻触屏幕中央区域** → 唤出工具栏

#### 唤出工具栏状态

```
┌────────────────────────────────────┐
│ ←  第3章 人物介绍           🔍  ⋮  │  ← 顶部栏：返回 + 章节名 + 搜索 + 更多
├────────────────────────────────────┤
│                                    │
│                                    │
│          [书籍内容]                 │
│                                    │
│                                    │
├────────────────────────────────────┤
│  ○━━━━━━━━━━━●━━━━━━━━━○          │  ← 进度条（可拖动）
│  12%        第3章         共12章   │
├────────────────────────────────────┤
│  📑     📝     🔊     💬     Aa  │  ← 底部工具栏
│ 目录   笔记   朗读    AI   设置   │
└────────────────────────────────────┘
```

**顶部栏**：
- `←` 返回书库
- 章节标题（居中）
- 🔍 全书搜索（点击展开搜索栏）
- `⋮` 更多菜单（向量化、书籍信息等）

**底部工具栏**（5 个操作）：
| 图标 | 功能 | 点击行为 |
|------|------|---------|
| 📑 目录 | TOC | 从底部弹出半屏 Sheet，显示可折叠目录树 |
| 📝 笔记 | NotebookPanel | 从底部弹出全屏 Sheet，显示笔记/高亮列表 |
| 🔊 朗读 | TTS | 开始朗读，底部显示迷你播放条 |
| 💬 AI | ChatPanel | 从底部弹出全屏 Sheet，嵌入 AI 对话 |
| Aa 设置 | ReadSettings | 从底部弹出半屏 Sheet，阅读样式设置 |

#### 选中文本弹出框

```
         ┌─────────────────────────┐
         │ 🟡 🟢 🔵 🩷 🟣 │ 📝 📋 🌐 🤖 🔊 │
         └─────────────────────────┘
              ▼
    ╔═══════════════════════════╗
    ║   用户选中的一段文字内容    ║
    ╚═══════════════════════════╝
```

- 与桌面端功能一致：高亮颜色选择 + 笔记 + 复制 + 翻译 + 问AI + 朗读
- 弹出框自适应在选中文本的上方或下方
- 针对移动端：按钮间距加大，确保触摸精度（最小 44px 触摸区域）

## 九、组件与样式一致性方案

移动端与桌面端保持同一套设计语言和组件 API，确保视觉和交互的统一性。

### 9.1 共享技术栈

| 层 | 桌面端 | 移动端 | 说明 |
|---|--------|--------|------|
| CSS 框架 | Tailwind CSS v4 | **同** | `@import "tailwindcss"` + `@tailwindcss/vite` 插件 |
| 主题系统 | oklch CSS 变量 + `@theme inline` | **同** | 两端 globals.css 颜色变量完全一致 |
| 类名工具 | `cn()` from `@readany/core/utils` | **同** | `clsx` + `tailwind-merge` |
| 变体管理 | `class-variance-authority` (CVA) | **需补齐** | Button、Badge 等组件的 variant 定义 |
| 无头组件 | Radix UI Primitives | **需补齐** | Dialog、Switch、Select、Slider 等 |
| 图标库 | `lucide-react` | **同** | 已安装 |
| Toast | `sonner` | **同** | 已安装 |

### 9.2 移动端需补齐的依赖

### 9.3 共享 UI 组件清单

移动端从桌面端**复制**以下 `components/ui/` 基础组件，保持 **相同的组件 API 和 variant 定义**，仅在 Tailwind 类名上做移动端微调：

| 组件 | 来源 | 移动端调整 |
|------|------|-----------|
| `button.tsx` | 桌面端 1:1 | 增大触摸区域：`size.default` 改为 `h-11 px-4 py-3`，`size.icon` 改为 `size-11` |
| `badge.tsx` | 桌面端 1:1 | 无调整 |
| `input.tsx` | 桌面端 1:1 | 高度改为 `h-11`，字号 `text-base`（避免 iOS 自动缩放） |
| `textarea.tsx` | 桌面端 1:1 | 字号 `text-base` |
| `switch.tsx` | 桌面端 1:1 | 无调整（Radix 原语） |
| `slider.tsx` | 桌面端 1:1 | Thumb 增大至 `h-6 w-6`，Track 增高至 `h-2` |
| `select.tsx` | 桌面端 1:1 | 触发器高度 `h-11`，字号 `text-base` |
| `dialog.tsx` | 桌面端 1:1 | 移动端改为底部弹出 Sheet 样式（见下方 9.4） |
| `dropdown-menu.tsx` | 桌面端 1:1 | 菜单项增大至 `py-3`，`text-base` |
| `tooltip.tsx` | 桌面端 1:1 | 移动端改为长按触发，或替换为内联文字标签 |
| `collapsible.tsx` | 桌面端 1:1 | 无调整 |

### 9.4 移动端新增组件

这些是移动端特有的，桌面端没有：

### 9.5 globals.css 对齐策略

移动端 `globals.css` 已与桌面端**共享完全相同的 CSS 变量**（oklch 色彩 + 阴影 + 圆角 + 字体），额外添加：

- `env(safe-area-inset-*)` 变量
- `100dvh` / `100dvw` 动态视口单位
- 移动端特有样式（`.chat-markdown` 在移动端需要时按需引入）

**原则**：颜色 token、圆角 token、字体 token 两端严格一致，仅在间距和字号上做移动端适配（更大触摸区、text-base 避免 iOS 缩放）。

## 十、页面路由结构
